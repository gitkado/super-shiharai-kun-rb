# AI開発ディレクトリ（`ai/`）

Claude Codeの `/dev` と `/verify` コマンドが使用する開発支援ディレクトリ。

## ディレクトリ構造

```text
ai/
├── board.md              # 作業ボード（現在の実装状況）
├── lessons.md            # 教訓・改善ルール（自己改善ループ）
├── specs/                # 機能仕様（永続保存）
│   └── <feature>/
│       ├── requirements.md
│       ├── design.md
│       └── tasks.md
└── README.md             # 運用ガイド
```

## Task機能との併用

Claude CodeのTask機能（TaskCreate, TaskUpdate, TaskList, TaskGet）と`ai/`ディレクトリは**役割分担**して併用する。

| 管理対象 | 担当 | 理由 |
|----------|------|------|
| 実行管理（状態遷移・依存関係・担当） | Task機能 | リアルタイムの進捗追跡に最適 |
| 統合ビュー（全体像・優先度・ログ） | board.md | 意思決定の背景・文脈を保存 |
| 永続記録（仕様・設計・根拠） | specs/ | 変更理由を後から参照可能 |

**注意**: Task機能はセッション単位で管理される。長期プロジェクトの記録は `specs/<feature>/tasks.md` に永続化する。

## タブ分離（並列作業）

| タブ | コマンド | 役割 | 編集権限 |
|------|----------|------|----------|
| Dev | `/dev` | 設計・実装・コミット | あり |
| Verify | `/verify` | テスト・レビュー | なし（報告のみ） |

詳細ガイド: `ai/README.md`

## Planモード活用基準

- 3ステップ以上のタスク → Planモード必須
- アーキテクチャ変更・新パッケージ追加 → Planモード必須
- 単純なバグ修正・1ファイル変更 → 直接実行OK
- 判断に迷う場合 → Planモードを選択（過剰な計画 < 手戻り）

## サブエージェント活用方針

- 調査・リサーチタスク → Exploreエージェントに委譲
- TDD実装 → tdd-executorに委譲
- 検証パイプライン → verifierに委譲
- SmartHR UI準拠チェック → ds-checkerに委譲
- メインコンテキストは**意思決定と統合**に集中し、詳細な調査で汚さない

## 自己改善ループ

- セッション開始時に `ai/lessons.md` をレビューする
- ユーザーからの修正フィードバックを受けたら、ミスパターンと予防ルールを `ai/lessons.md` に記録する
- 同じカテゴリのミスが繰り返される場合、`rules/` に恒久的なルールとして昇格を検討する
